# Human-in-the-Loop Governance

## Philosophy

Enterprise trust requires human governance.

The platform exists to:

- augment teams
- accelerate analysis
- reduce entropy

Not replace architectural accountability.

---

## Governance Workflow

1. Discovery
2. Analysis
3. Recommendation
4. Human review
5. Controlled deployment
6. Validation
7. Audit logging

---

## Mandatory Controls

### Required Human Approval

Required for:

- SQL deployment
- semantic model changes
- gateway changes
- refresh modifications

---

## Auditability

All actions must be:

- logged
- explainable
- reversible
- attributable

---

## Explainability Requirements

Every recommendation must explain:

- why it was generated
- expected impact
- impacted assets
- confidence level
- rollback considerations
